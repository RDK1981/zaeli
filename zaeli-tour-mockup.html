Build notes — v2 simplified

  • One template, eleven stops — same dedicated-route layout for everything: header + eyebrow + h1 + sub + progress bar + card + bottom nav. No special tour styling inside live sheets.
  • Tour card structure — icon, title, sub, "Try saying" callout (mint dashed border), feature pills, primary CTA. Tutor adds: trial badge, secondary CTA, price line.
  • "Try saying" callout — natural-language hook on every stop. Reinforces chat-first without elaborate machinery. Sky-blue variant for tap-actions (Photos).
  • First-time sheet banner — only on first open during tour. Mint banner inside the live sheet. Dismissable. Stops users bouncing out without trying anything.
  • Tutor as hero (Stop 7) — between daily loop and More Features. Violet accent throughout (icon ring, eyebrow, h1 highlight, CTAs, progress fill). Trial badge top-right of card. Two CTAs.
  • Tour pill — floats bottom-right during 14-day trial only. Shows days left (post-tour) or progress (mid-tour). Disappears post-trial.
  • Order rationale — daily loop first (Shopping/Meals/Calendar/Kids/Tasks/Photos), then Tutor as the climactic reveal, then bonus modules (Travel/Budget/MySpace/Family).
  • State machine (lib/tour-state.ts) — getCurrentStop(), advanceStop(), completeTour(), replayStop(n), shouldRePrompt(). AsyncStorage backed.
  • Inactivity — if mid-tour and chat closed for 24h+, next morning brief is followed by "pick up where you left off" prompt with progress shown.
  • Replay forever — Settings → Replay tour stays accessible post-trial. Per-stop picker plus "Start full tour" hero CTA.
