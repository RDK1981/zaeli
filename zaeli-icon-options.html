Notes

Apple's icon rules — what the PNG must be

  • 1024×1024 PNG for App Store + 2048×2048 PNG for the splash / launch screen.
  • No alpha channel, no transparency, no rounded corners — iOS applies its own mask on top. Draw a solid square background in the exported PNG.
  • Background colour must be #FAF8F5 (warm bg). No gradients — flat colour reads best at small sizes.
  • All glyphs / accents must be embedded pixels — no reliance on OS fonts.
  • Expo consumes it as assets/icon.png (referenced in app.json). Adaptive Android icon is a separate deal but not blocking iOS TestFlight.

If you pick 2A

  • Once you say the word I'll write an exact SVG spec of 2A that a friend / designer can open in any vector tool and export to 1024×1024 PNG.
  • The SVG will encode exact colours (#FAF8F5 bg, #0A0A0A z, #A8D8F0 a), exact letter-spacing, exact glyph sizing — nothing to eyeball.
  • Time to ship: ~15 min from PNG in hand → dropped into assets/icon.png → next EAS build carries it.
