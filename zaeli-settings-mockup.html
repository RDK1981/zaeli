Build notes

Scope v1

  • Account hero — avatar, name, email, plan tag
  • Subscription card — plan, price, next bill, add-ons, manage
  • Family — single row, opens Our Family
  • Notifications — full sub-screen (frame 2)
  • Zaeli's memory — full sub-screen (frame 3)
  • Integrations — placeholder row, v2 scope
  • Data & Privacy — export, clear chat, privacy policy
  • About — help, rate, terms, version
  • Danger zone — sign out, delete account

Pattern rules

  • Standalone full-screen route — app/(tabs)/settings.tsx
  • Back arrow in header = router.back() to Chat
  • Hamburger opens MoreSheet (cross-nav consistency)
  • Accent colour #6B7280 slate (channel convention)
  • iOS-grouped lists: white cards, 16px radius, divider lines
  • Toggle = native iOS style, green #34C759 on
  • Row icon tile: 34×34, 10px radius, tinted bg per section

Data sources

  • Account — Supabase auth user + family_members where name = 'Rich'
  • Subscription — Stripe metadata (via backend when web sales live) · placeholder "Family · A$14.99" for now
  • Notifications toggles — new user_preferences table (user_id, key, value) OR AsyncStorage for v1
  • Zaeli's memory — reads family_insights + family_milestones + conversation_memory. Delete row = soft delete with confirmation
  • Integrations — future — Apple Calendar, Google Calendar, HealthKit

Sub-screens not mocked

  • Manage subscription — opens Stripe customer portal in WebView (post-launch)
  • Integrations — toggles for each integration + auth flow
  • Export your data — generates JSON/ZIP, shares via iOS share sheet
  • Clear chat history — confirmation modal, deletes messages rows
  • Privacy policy / Terms — WebView to zaeli.app/privacy
  • Help & support — opens email or in-app Zaeli chat with support context
  • Delete account — hard confirmation flow, then cascade delete

Open questions for Richard

  • Do we want a Voice preferences row (auto-mic on open, default language)?
  • Do we show the AI engine toggle in Settings (for internal testing) or keep it hidden?
  • Quiet hours — a single range OK or do we need weekday/weekend split?
  • Manage subscription — direct to Stripe or intermediate screen first showing plan details + change/cancel?
  • Profile photo — allow upload, or stick with initial avatar?

Supabase migration needed

  • user_preferences table (user_id, pref_key, pref_value, updated_at) — stores all toggles
  • Defaults: all notifications on, quiet hours 21:30-06:30, sound on, vibration off
  • family_insights — add is_dismissed column so user can hide without delete
