Option 1 — Two-tier stacked

Chat bar on top · FAB underneath · always visible

Existing chat bar sits above a full FAB bar. Zero code changes to the chat bar — we just show the FAB on chat page (currently hidden via activePage !== PAGE_CHAT).

Camera lives inside the TextInput as designed in Session 9. Mic also lives inside the chat bar. The FAB mic becomes redundant but harmless.

Pros

→ Zero risk to chat bar — untouched

→ Full access to Dashboard / My Space / More without swipe

→ Consistent with Dashboard & My Space screens

→ Camera + upload clearly available

Cons

→ 110-120px of bottom chrome (two bars)

→ Two mics (FAB mic + chat bar mic) — duplicated

→ Feels heavy; less "conversation-like"

→ Keyboard open pushes BOTH bars up, lots of movement

Option 1B — Two-tier, FAB auto-hides

Same as Option 1 but FAB collapses when chat input is focused

When Rich is reading the feed, the FAB is visible underneath the chat bar (full nav access). When he taps the chat input or starts recording, the FAB slides down out of view — only the chat bar stays. When keyboard dismisses, FAB slides back up.

Extra screen real estate when typing. Full navigation when not. Chat bar stays unchanged.

Pros

→ Clean chat experience when typing (no extra chrome)

→ Full nav when reading the feed

→ Chat bar stays untouched — safe

→ Animates in/out — feels polished

Cons

→ Needs Keyboard.addListener — but CLAUDE.md says never setState from this

→ Workaround: use onFocus/onBlur on TextInput but CLAUDE.md says no onBlur handler!

→ Can't easily drive FAB visibility from chat input state without breaking the rules

→ Risky — this is exactly the pattern that caused the 10-hour bug

Option 2 — Merged single bar

Nav + input + camera + mic + send + more all in one 48px bar

Every piece of functionality crammed into a single bottom bar: [▦ Dashboard] [Type a message…] [📷 Camera] [🎤 Mic] [➤ Send] [⋯ More]. Super compact — maximum one bar of chrome.

The chat input loses real estate — maybe 140px wide instead of the current ~260px. Placeholder text "Ask Zaeli…" gets truncated on small devices.

Pros

→ Absolute minimum chrome — one bar only

→ Everything accessible in one tap

→ Looks sleek at first glance

Cons

→ Breaks the untouched chat bar rule — this IS the chat bar with nav added

→ Text input becomes tiny — Rich has to scroll to see what he typed

→ Send button neighbours Mic and More — accidental taps highly likely

→ Five+ touch targets in a 340px wide bar = cramped, small hit areas

→ Hardest to fix if it breaks

Option 3 — Minimal chat bar + floating More button

Chat bar untouched · floating ⋯ top-right · swipe for Dashboard/My Space

The Chat screen is almost pure — just the feed and the existing chat bar. A single floating "⋯" circular button sits top-right (44px, soft shadow). Tapping it opens the full More sheet which already holds Dashboard, My Space, Tutor, Kids Hub, Our Family, Our Budget, Settings, plus now Camera/Upload.

Dashboard and My Space remain accessible via swipe gestures — that's the primary navigation. The More sheet is the "everything else" entry point.

Pros

→ Zero risk to chat bar — completely untouched

→ Cleanest chat experience — feels like a conversation, not an app

→ Everything still accessible (swipe + More sheet)

→ Brief bubble has maximum vertical space

→ Camera in both places — inside input (one-tap) AND in More sheet upload grid

→ Smallest code change

Cons

→ Floating button is a less-familiar pattern for navigation

→ No visible "Dashboard" / "My Space" labels — swipe is the only direct path

→ If Rich doesn't know swipe works, he's stuck in Chat

→ Loses the FAB's bottom-anchored muscle memory

What's in the More sheet

Opened by tapping the ⋯ floating button

Top section — Upload grid (3 columns):

  • 📷 Take photo · 🖼️ Choose from library · 📄 Upload file

Family channels (sheet overlays over Chat):

  • Calendar · Shopping · Meals · To-dos · Notes · Travel

Dedicated screens:

  • Tutor · Kids Hub · Our Family · Our Budget

At the bottom: Settings

This is essentially the existing FAB More sheet — just extended with the upload grid at the top and opened from a floating button instead of the FAB bar.

My recommendation

If you're unsure, here's where I'd land

Option 3 for the Chat screen. Chat is Zaeli's home — the most precious screen in the product. Keeping it minimal (just the brief, the conversation, and the one chat bar we've spent so long perfecting) is the right move. The floating ⋯ button gives everything else, one tap away.

For Dashboard and My Space, keep the existing FAB bar — those screens benefit from the 5-button bottom navigation because they're not conversations. They're reference layers where you need to jump around.

If you want to explore Option 1 instead (two-tier stacked FAB), we can — but only in its "always visible" form. The 1B auto-hide variant is the pattern that caused the 10-hour chat-bar bug. I'd avoid it.
