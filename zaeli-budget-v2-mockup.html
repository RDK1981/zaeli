Build notes — v2 pure planner

What's in

  • Income streams — same as before, per earner
  • Fixed categories with line items — user enumerates actual commitments (Netflix, Spotify, iCloud). Category total = SUM(line_items.monthly_amount), auto.
  • Variable categories — single editable monthly_target. No line items.
  • Savings goals — forward-looking, manual. User updates "saved so far" periodically.
  • Budget health — Budgeted % / Savings rate / Spare. Describes the plan, not performance.
  • One-off AI helper — "Help me set realistic budgets". Upload statements → Claude Sonnet vision → suggestions per category (accept/edit/skip). Uploads are not stored.

What's out

  • Live "spent this month" numbers — removed everywhere
  • Status tiles (on track / watch / over) — replaced by budget health tiles
  • Transaction ledger — no per-category "This month" list
  • Monthly reviews — no persisted spent-by-month rows
  • Date-stamped reality check banner
  • Category detail "Add transaction" button — no live ledger to add to

Data model

  • income_streams(id, family_id, member_id, label, type, monthly_amount)
  • budget_categories(id, family_id, name, emoji, type, monthly_target_variable, sort_order) — monthly_target_variable nullable for fixed
  • category_line_items(id, category_id, label, monthly_amount, sort_order) — only for fixed categories
  • savings_goals(id, family_id, name, emoji, target, saved_amount, monthly_contribution, target_date)
  • Effective category budget = variable? monthly_target_variable : SUM(line_items)

AI helper flow

  • Entry: Overview "Help me set realistic budgets" button · or per-category from detail sheet
  • Multi-image picker (up to 6 statement screenshots)
  • Claude Sonnet vision → averages spend by category across months, detects recurring subscriptions
  • Review sheet: one card per suggestion with Accept / Edit / Skip
  • On Save: write accepted amounts to budget_categories / category_line_items. Raw upload data discarded.
  • No transaction ledger created. No month persistence.

Zaeli's voice in-module

  • Setup nudges — "Want help setting realistic budgets?" when categories are empty
  • Plan quality commentary — "74% budgeted, 15% savings rate — strong plan"
  • Comparative nudges — "Groceries at $450 is low for 5 people. Typical range is $520-580."
  • Goal nudges — "$50 more per month finishes Noosa by July."
  • What-if (later) — pay rise, part time, remove car loan — pure math, no tracking

Open for feedback

  • "Savings" in income hero grid — is "Savings $1,400" derived from sum of goal contributions, or a manual target? I've shown sum-of-contributions.
  • Variable category tip about "spent this month" — worth including, or too much dev-speak surfaced to user?
  • What-if mode — still in scope for v1 or defer? Pure math, fits the planner vision.
  • Onboarding flow — first-time setup using the AI helper by default, or manual-first with AI as optional shortcut?
